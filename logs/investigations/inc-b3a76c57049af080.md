## Summary


Google Threat Intelligence Group (GTIG) identified a multistage intrusion campaign by a newly tracked threat group, UNC6692, that leveraged persistent social engineering, a custom modular malware suite, and deft pivoting inside the victim's environment to achieve deep network penetration.

## Technical Details

### Initial Access

UNC6692 relied heavily on impersonating IT helpdesk employees, convincing their victim to accept a Microsoft Teams chat invitation from an account outside their organization. In late December 2025, UNC6692 conducted a large email campaign designed to overwhelm the target with messages, creating a sense of urgency and distraction.



The victim is then manipulated into clicking a phishing link shared via the Teams chat disguised as a "Mailbox Repair and Sync Utility v2.1.5," which triggers the download of a malicious AutoHotkey script from an attacker-controlled Amazon S3 bucket.


**MITRE ATT&CK**: T1566.002 (Spearphishing Link), T1204.001 (Malicious Link)

### Phishing Landing Page (Anti-Analysis)
The Mandiant report details a sophisticated gatekeeper script on the phishing page that: (1) checks for a mandatory `?email=` URL parameter, redirecting to `about:blank` if absent; (2) forces victims into Microsoft Edge via the `microsoft-edge:` URI scheme; and (3) employs a "double-entry" credential harvesting trick that rejects the first two password attempts to ensure accurate capture. 
Victims are directed to a spoofed "Mailbox Repair Utility" page, where they are prompted to enter credentials intentionally rejected on the first attempt to ensure password capture before exfiltration to an attacker-controlled AWS server.


### CVEs
**None.** No CVEs are referenced in the Mandiant report or any corroborating source. The attack relies entirely on social engineering and abuse of legitimate tools.

### SNOW Malware Ecosystem


The SNOW malware ecosystem, attributed to the threat cluster UNC6692, operates as a modular ecosystem comprising three primary components: SNOWBELT, SNOWGLAZE, and SNOWBASIN. Rather than functioning as isolated tools, these components form a coordinated pipeline that facilitates an attacker's journey from initial browser-based access to the internal network of the organization.


**1. SNOWBELT (Browser Extension)**

SNOWBELT serves as the initial foothold and the primary "eyes" of the operation. It is a JavaScript-based backdoor delivered as a Chromium browser extension, often masquerading under names like "MS Heartbeat" or "System Heartbeat." Rather than being available through the Chrome Web Store, the extension is deployed through social engineering tactics.
 It uses a time-based DGA with a hard-coded seed to calculate C2 URLs on AWS S3, and leverages AES-GCM encryption for C2 communications. It also registers with the browser's Push Notification service for low-latency, asynchronous C2 wake-up. **MITRE**: T1176.001 (Browser Extensions), T1547.001 (Startup Folder)

**2. SNOWGLAZE (Python Tunneler)**

SNOWGLAZE is a Python-based tunneler that runs in both Windows and Linux environments and manages external communication. It creates an authenticated WebSocket tunnel between the victim's internal network and the attacker's C2 infrastructure, such as a Heroku subdomain.
 It masks traffic by wrapping data in JSON objects with Base64 encoding, mimicking standard encrypted web traffic. Hard-coded C2: `wss://sad4w7h913-b4a57f9c36eb[.]herokuapp[.]com:443/ws`. **MITRE**: T1572 (Protocol Tunneling), T1090 (Proxy)

**3. SNOWBASIN (Python Backdoor)**
SNOWBASIN is a Python-based backdoor operating as a local HTTP server (ports 8000–8002). It supports remote shell access (`/stream`), file exfiltration (`/buffer`), file ingress (`/commit`), screenshot capture (`/capture`), and self-termination (`/gc`). Commands are routed through SNOWGLAZE and relayed by SNOWBELT via HTTP POST to `localhost:8000`. **MITRE**: T1059.006 (Python), T1113 (Screen Capture), T1005 (Data from Local System)

### Post-Compromise TTPs


After gaining initial access, UNC6692 used a Python script to scan the local network for ports 135, 445, and 3389. Following internal port scanning, the threat actor established a Sysinternals PsExec session to the victim's system via the SNOWGLAZE tunnel, and executed commands to enumerate local administrator accounts. Using the local administrator account, the threat actor initiated an RDP session via the SNOWGLAZE tunnel from the victim system to a backup server.
 **MITRE**: T1046 (Network Service Discovery), T1021.001 (RDP), T1021.002 (SMB)


The threat actor utilized a local administrator account to extract the system's LSASS process memory with Windows Task Manager for privilege escalation.
 **MITRE**: T1003.001 (LSASS Memory)


Armed with the password hashes of elevated users, UNC6692 used Pass-The-Hash to move laterally to the network's domain controllers. Once authenticated to the Domain Controller, the threat actor downloaded a ZIP archive containing FTK Imager to the Domain Administrator's `\Downloads` folder. The threat actor executed FTK Imager and mounted the local storage drive. Subsequently, FTK Imager wrote the Active Directory database file (NTDS.dit), Security Account Manager (SAM), SYSTEM, and SECURITY registry hives to the `\Downloads` folder. The extracted files were then exfiltrated from the network via LimeWire.
 **MITRE**: T1003.003 (NTDS), T1003.002 (SAM), T1567.002 (Exfiltration to Cloud Storage)

### "Living Off the Cloud" Strategy

A key element of UNC6692's approach is routing both payload delivery and data exfiltration through trusted cloud services like Amazon S3. By leveraging legitimate platforms, the attackers can blend into normal cloud traffic and bypass traditional reputation-based network filters, making detection significantly harder.
